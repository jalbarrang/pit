## Decoder choice

Chosen: `upng-js` (MIT, one `pako` dependency) + `jpeg-js` (BSD-3-Clause, no dependencies). `jimp` was rejected because `pnpm view` showed a ~3.3MB unpacked package with many plugins/transitive packages for formats pit does not need now. The chosen pair is pure JS, has no native postinstall, and matches the current deliverable: PNG/JPEG bytes to RGBA for `OptimizedBuffer.drawSuperSampleBuffer`.

## Split-footer/passthrough spike

Spike command: `scripts/dev.sh /tmp/pit-split-footer-spike.ts passthrough|capture` under `script(1)`, using `createCliRenderer({ screenMode: "split-footer", footerHeight: 6, externalOutputMode })` plus a tiny footer root. Observations: `passthrough` keeps incidental stdout in the real terminal path, matching opencode's `app.tsx`, but pit's existing `ScrollBox` chat remains inside the 6-row footer and does not become real scrollback automatically; a straight config flip would shrink the app and worsen history browsing. `capture-stdout` enables opentui's `writeToScrollback`/`createScrollbackSurface` API, so committed rows can be replayed above the footer, but that requires a structural pivot: split pit into an append-only transcript writer plus a footer editor/chrome, and stop treating chat as one fullscreen `ScrollBox`. Recommendation: keep fullscreen for now; revisit split-footer only as a signed-off redesign when native terminal scrollback/copy is more valuable than the current integrated scroll viewport, mouse routing, overlays, and extension slots.
